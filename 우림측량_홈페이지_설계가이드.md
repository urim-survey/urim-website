# 우림측량 홈페이지 설계 가이드라인

> 레퍼런스(건축사사무소 시월 / siwolarchi.com)의 미니멀·편집형 레이아웃을
> **우림측량의 네이비 신뢰형 브랜드**로 변환한 상세 설계서입니다.
> Claude Code가 이 문서를 그대로 읽고 페이지를 구현할 수 있도록 작성했습니다.

---

## 0. 확정된 설계 방향

| 항목 | 결정 |
|------|------|
| 디자인 방향 | **절충형** — 시월의 미니멀·여백 중심 레이아웃 + 우림측량 네이비 컬러 |
| 메인 핵심 섹션 | **핵심 서비스 3종** (개발행위허가 / 농지전용허가 / 태양광 인허가) |
| 이미지 전략 | **AI 생성 이미지** (DALL·E)로 대체 — 본문 7장 프롬프트 제공 |
| 톤 | 전문적·신뢰감 (작품 전시형 ❌ → 상담 유도형 ⭕) |
| 주요 목표 | 구글 SEO + 상담 문의 전환(리드 확보) |

### 레퍼런스에서 가져올 것 / 버릴 것

**가져올 것 (Keep)**
- 넓은 여백과 큰 타이포로 만드는 '여유로운 전문가' 인상
- 영문 섹션 라벨(`OUR SERVICES`, `WORK PROCESS`)로 만드는 편집디자인 감성
- 풀블리드(full-bleed) 히어로 이미지 + 짧은 슬로건
- 카드형 정보 정리, 하단 문의 유도 흐름

**버릴 것 (Drop)**
- 흑백 무채색 → 네이비/블루 + 화이트로 대체
- '작품 감상' 중심의 시적인 카피 → 명확하고 신뢰감 있는 실용 카피
- 포트폴리오(작품 사진) 중심 구조 → 서비스·신뢰·절차 중심 구조

## 1. 디자인 시스템

### 1-1. 컬러 토큰

```css
/* Brand */
--navy:        #003087;  /* 메인 — 헤더, 강조 텍스트, 푸터 */
--blue:        #0057b8;  /* 보조 강조 — 버튼 호버, 링크, 포인트 */
--blue-light:  #E8EEF7;  /* 연한 배경 블록, 카드 호버 */

/* Neutral */
--ink:         #1A1A1A;  /* 본문 기본 텍스트 */
--gray-700:    #4A4A4A;  /* 보조 본문 */
--gray-400:    #9AA0A6;  /* 캡션, placeholder */
--line:        #E5E7EB;  /* 구분선, 카드 테두리 */
--bg:          #FFFFFF;  /* 기본 배경 */
--bg-soft:     #F7F8FA;  /* 섹션 교차 배경 */

/* State */
--success:     #2E7D32;
--cta:         #003087;  /* 주 버튼 = 네이비 */
```

> 규칙: 네이비는 **신뢰·구조**, 블루는 **행동 유도(버튼·링크)**에 사용. 한 화면에 색을 3개 이상 쓰지 않습니다.

### 1-2. 타이포그래피

- **한글 본문/제목**: `Noto Sans KR` (Google Fonts)
- **영문 섹션 라벨**: `Inter` 또는 시스템 산세리프 — letter-spacing 넓게(편집 감성)

```
H1 (히어로)      : 40~56px / 700 / line-height 1.2
H2 (섹션 제목)   : 28~36px / 700
H3 (카드 제목)   : 20~22px / 600
본문             : 16~17px / 400 / line-height 1.7
영문 라벨        : 13~14px / 600 / letter-spacing 0.15em / 대문자
캡션            : 13px / 400 / --gray-400
```

### 1-3. 레이아웃 · 여백

```
컨테이너 최대폭   : 1200px (본문), 히어로는 full-bleed
섹션 상하 패딩    : 데스크탑 96~120px / 모바일 56~64px
카드 간격(gap)    : 24~32px
그리드           : 12컬럼 / 모바일 1컬럼
모서리 둥글기     : 8px (카드), 4px (버튼)
그림자           : 은은하게 1단계만 (0 2px 12px rgba(0,0,0,0.06))
```

> 시월 레퍼런스의 핵심은 **여백**입니다. 요소를 빽빽하게 채우지 말고, 섹션 사이를 충분히 비워 '전문가의 여유'를 연출합니다.

### 1-4. 버튼

| 종류 | 스타일 |
|------|--------|
| Primary | 배경 `--navy`, 흰 글씨, 호버 시 `--blue` |
| Secondary | 테두리 `--navy`, 글씨 `--navy`, 호버 시 배경 연하게 |
| 전화 CTA | 아이콘 + 전화번호, 모바일에서 항상 노출 |

## 3. 홈(`/`) 상세 설계 — 레퍼런스 구조 변환

레퍼런스 홈은 `Hero → FEATURED PROJECTS → RECENT NEWS → CONTACT` 흐름입니다.
이를 우림측량용으로 아래처럼 변환합니다.

### 섹션 ① Hero (풀블리드)
- 배경: AI 이미지(드론에서 본 토지/측량 현장, 네이비 오버레이 40%)
- 영문 라벨: `URIM SURVEY` (상단 작게)
- H1 슬로건(예시, 수정 가능):
  > **"정확한 측량, 확실한 인허가"**
  > 횡성 토지의 모든 절차를 우림측량이 책임집니다.
- CTA 2개: `상담 문의하기`(Primary) / `전화 상담`(Secondary, 전화번호)

### 섹션 ② 핵심 서비스 3종 ★ (← FEATURED PROJECTS 자리)
- 영문 라벨: `OUR SERVICES`
- 3개 카드 가로 배치(모바일 세로):

| 카드 | 제목 | 한 줄 설명 |
|------|------|-----------|
| 1 | 개발행위허가 | 토지 개발을 위한 인허가 전 과정 대행 |
| 2 | 농지전용허가 | 농지를 대지·잡종지로 전환하는 절차 |
| 3 | 태양광 인허가 | 태양광 발전소 설치 인허가 일괄 처리 |

- 각 카드: AI 아이콘/이미지 + 제목 + 1~2줄 설명 + `자세히 보기`(→ /services 해당 섹션)

### 섹션 ③ 우림측량을 선택하는 이유 (신뢰 블록)
> 측량·인허가업은 '작품'이 아닌 '신뢰'를 팝니다. 레퍼런스에 없던 신뢰 섹션을 추가합니다.
- 배경: `--bg-soft`
- 3~4개 신뢰 포인트(아이콘 + 숫자/문구):
  - 횡성 지역 토지 전문 / 풍부한 인허가 경험 / 신속·정확한 처리 / 처음부터 끝까지 1:1 관리
  - (실제 경력·실적 수치는 §8 확인 필요 항목)

### 섹션 ④ 업무절차 미리보기 (← /guide 요약)
- 영문 라벨: `WORK PROCESS`
- 5단계를 가로 타임라인으로 요약 + `전체 절차 보기`(→ /guide)
  - 상담 → 현장확인 → 서류준비 → 인허가 신청 → 완료 (각 단계 예상 기간 표기)

### 섹션 ⑤ 최근 소식 (← RECENT NEWS 변환)
- 영문 라벨: `RECENT NEWS`
- Supabase `blog_posts`에서 최신 3개 카드(썸네일+제목+날짜) + `블로그 더보기`(→ /blog)

### 섹션 ⑥ 하단 상담 유도 (CTA 밴드)
- 배경: `--navy` 풀블리드
- 흰 글씨 문구 + `상담 문의` 버튼 + 전화/카톡 아이콘

### 푸터
- 업체명/대표/주소/전화/사업자번호 + 메뉴 + 운영시간(평일 09:00~19:00) + 저작권

## 4. 나머지 페이지 설계

### 4-1. 회사소개 `/about`
- 히어로(작은 배너) + 영문 라벨 `ABOUT`
- 회사 철학/인사말(신뢰감 있는 톤, 시적X)
- 대표/조직 소개 (레퍼런스의 '대표 프로필' 차용)
- 보유 장비·기술·자격(측량업 등록증 등) → 전문성 근거
- 오시는 길(구글지도 — 횡성군 태기로, 정확 주소 입력 시 적용)

### 4-2. 서비스안내 `/services`
- 3대 서비스를 각각 한 섹션씩 상세 설명
- 각 서비스: 정의(쉬운 설명) → 대상 → 필요 서류 → 진행 방식 → CTA
- 전문용어는 괄호로 쉽게 풀어쓰기 (의뢰인 눈높이)

### 4-3. 업무절차 `/guide`
- 5단계 세로 타임라인 (각 단계: 설명 + 예상 기간 + 의뢰인 준비물)
- 단계별 자주 막히는 부분 팁

### 4-4. 블로그 `/blog`
- 카드 그리드(썸네일+제목+카테고리+날짜)
- 카테고리 필터: `업무사례` / `업계소식` / `공지사항`
- 데이터: Supabase `blog_posts`, 관리자 작성은 `/admin/blog`(TipTap)

### 4-5. FAQ `/faq`
- 아코디언(접고 펼치기) 형식
- 예상 질문: 측량 비용은? / 인허가 기간은? / 농지전용 가능 여부 확인은? / 태양광 설치 조건은? / 상담은 무료인가요?

### 4-6. 문의상담 `/contact`
- 레퍼런스의 `A/T/H/E` 연락처 블록 차용 → 네이비 스타일로
- 연락처: 전화 / 카카오톡 / 이메일 / 주소
- 문의폼(이름·연락처·문의유형·내용) → **Supabase 저장**
- 구글지도 임베드

## 6. 전역 요소

### 헤더 (모든 페이지 상단 고정)
- 좌: 로고(우림측량) / 우: 메뉴 7개 + `상담문의` 버튼(네이비)
- 모바일: 햄버거 메뉴
- 스크롤 시 흰 배경 + 그림자로 고정

### 플로팅 버튼 (모바일 우하단 고정)
- 전화 / 카카오톡 빠른 연결 (전환율 핵심)

### 푸터 (모든 페이지 하단)
- 업체 정보 + 메뉴 + 운영시간 + 저작권

## 7. SEO 설계

- 페이지별 `title` / `meta description`에 **지역명(횡성) + 서비스명** 포함
- `sitemap.xml`, `robots.txt` 생성
- 구조화 데이터: `LocalBusiness` 스키마(상호·주소·전화·영업시간)
- 블로그 글: `지역명 + 서비스명` 롱테일 키워드 (예: "횡성 농지전용 절차")
- 이미지 `alt` 텍스트 필수
- Open Graph 태그(카톡/페북 공유 시 미리보기)
- 구글 서치콘솔 + 네이버 서치어드바이저 등록

## 9. 구현 순서 제안 (Claude Code용)

```
1. 디자인 시스템 세팅 (Tailwind config에 컬러/폰트 토큰)
2. 전역 레이아웃 (헤더/푸터/플로팅 버튼)
3. 홈(/) 6개 섹션 — placeholder 텍스트·이미지로 먼저 골격
4. 나머지 6개 페이지 골격
5. Supabase 연동 (블로그 + 문의폼)
6. AI 이미지 교체 → 실제 콘텐츠 입력
7. SEO 메타·sitemap → 배포 → 서치콘솔 등록
```
