#### 블로그 시스템 전체 재구축 (Supabase → Git 파일 기반)
- 글 저장 방식을 Supabase DB → 저장소 안 마크다운 파일(`content/posts/*.md`)로 전환
- 대표·본문 이미지도 Supabase Storage → `public/blog-images/` 폴더로 전환 (저장소에 같이 커밋됨)
- `/blog`, `/blog/[slug]`를 완전 정적 페이지로 전환 (revalidate 제거, `generateStaticParams` 사용) → 방문자가 봐도 Vercel 함수 호출 0번
- 글쓰기(`/blog/write`)·수정(`/blog/[slug]/edit`)은 **로컬(`npm run dev`)에서만 동작**, 배포 사이트에서는 완전 차단(404) — 글 작성 후 `git push`해야 사이트에 반영됨
- `src/lib/posts.ts` (파일 기반 CRUD), `/api/posts`, `/api/posts/[slug]`, `/api/posts/upload-image` 라우트 신규 추가 (전부 로컬 전용)

#### Vercel 사용량 절감
- `next.config.ts`에서 이미지 최적화(Image Optimization) 끔 (`unoptimized: true`) — 무료 요금제에서 가장 빨리 소진되는 항목이라 미리 차단

#### 문의상담 → 전화상담으로 전면 교체
- 기존 문의폼(이름·연락처·문의유형·내용 입력) 완전히 제거
- "전화 상담하기"(검정) + "카카오톡 상담하기"(카카오 브랜드 컬러 `#FEE500`) 버튼으로 대체
- 카카오톡 채널은 아직 미개설 — 채널 만드신 후 URL을 `src/app/contact/page.tsx`의 `KAKAO_CHANNEL_URL` 상수에 넣기만 하면 연결됨
- 구글 지도 iframe 임베드 추가 (API 키 필요 없는 방식, 비용 없음)

#### Supabase 완전 제거
- `/api/contact`, `src/lib/supabase.ts`, `src/lib/supabase-server.ts` 삭제
- `@supabase/supabase-js` 패키지 제거 (프로젝트에서 Supabase 의존성 없음)
- 기존 Supabase `posts`/`contacts` 테이블과 `blog-images` 버킷 데이터는 남아있지만 더 이상 코드에서 사용 안 함 — 필요 없으면 Supabase 콘솔에서 직접 정리 가능
